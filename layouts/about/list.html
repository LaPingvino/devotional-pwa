Methodology

Holy Writings brings together Bahá’í prayers and sacred texts from multiple online sources, cross-referenced using the Phelps Index — a catalog of 29,000+ Bahá’í Holy Writings identified by unique PIN codes (e.g. BH00074BLE for “Blessed is the spot”).

The matching pipeline

Each prayer or text entry from a source needs to be matched to its Phelps code. This is done through a multi-stage pipeline:

  1. Rosetta matching — If the same prayer already exists in another language with a known code, we match via cross-language reference tables (the “rosetta” approach). This is the most reliable method.
  2. Text similarity — For English prayers, we compare against the Phelps inventory using Jaccard word overlap, prefix matching, and length ratio analysis. Threshold: 0.45+ similarity score.
  3. LLM-assisted matching — When automated methods fail, we use Gemini Flash Lite to translate short excerpts and match against the inventory. This is flagged in the database as llm-translation source.
  4. Manual verification — High-value or ambiguous matches are reviewed by hand. False positives from LLM matching are tracked and blacklisted to prevent recurrence.

Devotional Suggestions

The Devotional Program builder offers thematic prayer suggestions based on the Bahá’í calendar and common devotional themes. Here’s how they work:

How suggestions are selected

Each Bahá’í month, holy day, and theme is associated with a set of prayer categories from bahaiprayers.net (e.g. the month of Núr / Light maps to “Enlightenment”, “Guidance”, and “Teaching”). Prayers matching those categories are then offered as suggestions.

  • Suggestions always draw from the English prayer catalog as the canonical source (for consistent selection across languages), but display text in your selected language when a translation exists.
  • Prayers are sorted by number of available translations, so the most widely translated prayers appear first.
  • Prayers flagged as llm-translation source are hidden from suggestions to avoid showing machine-generated text.

Limitations

  • Categories come from bahaiprayers.net and may not perfectly reflect each prayer’s content (see Known Issues).
  • The month–theme mapping is a starting point, not a definitive guide. Some associations are stronger than others.
  • Holy day suggestions rely on the Gregorian calendar date, which may not exactly match the Badí’ calendar holy day for all years (particularly the Twin Holy Days whose Gregorian dates shift annually).

Known Issues

This is a work in progress. Known data quality issues include:

Matching accuracy

  • LLM-assisted matches may contain errors. Approximately 5% of matches were made with LLM assistance. While most are correct, some may be false positives — especially for short prayers or prayers from the same tablet with similar openings.
  • TMP codes indicate prayers we couldn’t identify. There are currently ~46 unique TMP codes remaining across various languages. Some are composite entries (multiple prayers combined), some are regional texts not in the Phelps inventory, and some are non-prayer content (tablet excerpts, historical texts).
  • Category assignments come from source sites and may be incorrect. We’ve found and fixed several (e.g. the Tablet of Wisdom filed under “Children”), but more likely remain.

Source-specific issues

  • bahaiprayers.net French “Prayers & Meditations” was actually a different French prayer compilation, not the Shoghi Effendi selection. Purged; will be reimported from first principles.
  • Intro/front-page prayers on bahaiprayers.net (like “Blessed is the spot” and “Intone, O My servant”) are not indexed as prayers by the site, so they were missing from major languages. Partially recovered from bahaiprayers.app.
  • Some API language codes are wrong (e.g. Esperanto and Tagalog were mapped to English). We use the database’s languages.api_id field as the authoritative mapping.

Structural issues

  • Same-tablet prayers may share a Phelps code when they should have distinct mnemonic sub-codes. Length-ratio analysis (comparing the same code across languages) helps catch these.
  • Composite entries — some source sites combine multiple short prayers into one entry. These need splitting and individual coding.
  • Prayer vs. tablet excerpt — some entries categorized as “prayers” are actually excerpts from tablets or historical texts, not standalone prayers.

How You Can Help

This project benefits enormously from community input. Here’s how you can contribute:

Report errors

  • Wrong prayer code: If a prayer is showing up under the wrong Phelps code, or if two different prayers share the same code, please let us know the language, the prayer text (or first line), and what you believe the correct code should be.
  • Wrong category: If a prayer appears in the wrong category (e.g. a healing prayer filed under “Marriage”), report the language, Phelps code, current category, and correct category.
  • Missing prayers: If you know of a prayer that should be in a particular language but isn’t, let us know.
  • Text errors: Typos, formatting issues, or incorrect text in any language.

Help with identification

  • TMP codes: If you recognize a prayer currently tagged with a TMP code, tell us what it is! Visit the TMP entries in the catalog to browse them.
  • Regional prayers: Some prayers exist only in specific regions and may not be in the Phelps inventory. If you can identify the author or source, that helps us assign the correct code.

Translate the interface

The UI currently supports 13 languages. If you’d like to help translate the interface into another language, or improve an existing translation, please get in touch.

Technical contributions

The source code is available on GitHub. The prayer database is hosted on DoltHub and can be cloned and queried with SQL.
